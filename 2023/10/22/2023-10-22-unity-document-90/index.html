翻译Unity中文版的初衷是因为官方提供的中文版存在缺陷，而且翻译的不全。现在基于Unity2023.2版本对官方文档进行翻译。

Package Manager 🔗

要访问Package Manager设置，请打开Edit菜单，然后选择Project Settings，然后选择 Package Manager 类别。这些设置可以让您管理范围注册表并更改当前项目的高级设置。

(A) 在高级设置部分，切换预发布包在 Package Manager（包管理器）窗口中是否可见。

(B) 在 Scoped Registries（作用域注册表） 组下，添加、更改和删除项目的范围注册表，其中包含以下区域：

(1) 当前为该项目定义的作用域注册表列表。

(2) 所选作用域注册表的详细信息。

高级设置 🔗

使用高级设置组来切换当前项目的启用预发布包设置。

启用预发布包 🔗

启用启用预发布包设置可以在 Package Manager（包管理器）窗口中显示预发布包。预发布包在列表面板中显示 Pre 标签，详细面板中显示 Pre-Release 标签。这些标签用于提醒您哪些包将在下一个长期支持（LTS）版本的 Unity 编辑器中发布。

警告: 默认情况下，此设置被禁用，因为预发布包仍在验证稳定性是否足以用于生产，但尚未完全发布。有关包的生命周期的更多信息，请参阅 Package states and lifecycle。

Scoped Registries（作用域注册表） 🔗

作用域注册表组显示了当前项目中添加的作用域注册表的列表，列表位于设置组的左侧，并显示所选注册表的详细信息。

有关作用域注册表的详细信息，包括如何在 Unity Package Manager 中使用和配置它们，请参阅 Scoped Registries。

注意: 在添加作用域注册表时，您可能会遇到一个问题，即作用域注册表未列在 My Registries（我的注册表）上下文中，或者根本没有My Registries（我的注册表） 上下文可用。这可能是因为您添加的包注册表服务器未实现/-/v1/search或/-/all端点。Unity 的包管理器需要这些端点。

添加新的注册表 🔗

要向项目中添加新的作用域注册表，请按照以下步骤进行操作：

  1. 在作用域注册表列表底部选择 +按钮。一个新条目将出现在列表中，该条目名称为New Scoped Registry，右侧的详细信息为空。
  2. 输入名称、URL 和作用域属性的值。
  3. 如果需要指定多个作用域，请在最后一个作用域的值下方选择 + 按钮。将会出现另一个文本框。
  4. 输入所选作用域注册表的信息后，单击Save保存。要取消添加新的作用域注册表，请单击Cancel。

修改现有注册表 🔗

要修改现有的作用域注册表，请按照以下步骤进行操作：

  1. 在左侧的列表中选择要修改的注册表。现有的信息将显示在右侧。
  2. 更改名称、URL 和作用域属性中的任何一项。
  3. 在更新所选作用域注册表的信息后，单击Apply以应用更改。要取消更新，单击Revert。

删除注册表 🔗

要删除现有的作用域注册表，请按照以下步骤进行操作：

  1. 从列表中选择要删除的注册表。
  2. 单击列表下方的 - 按钮。出现一个对话框询问您确认删除。
  3. 单击Ok以删除注册表，或单击Cancel保留注册表。
