Visual Studio C# 集成 🔗

Unity 通过 Visual Studio 的 Code Editor Package 与 Microsoft Visual Studio 进行集成。这个包在安装 Unity 时预先安装。如果在安装 Unity 时已经安装了 Visual Studio，那么 Unity 默认使用 Visual Studio 来打开和编辑脚本。

将 Visual Studio 设置为默认的脚本编辑器 🔗

如果在安装 Unity 时安装了 Visual Studio，或者在 Unity 安装过程中安装了 Visual Studio，Unity 会自动将 Visual Studio 设置为默认的脚本编辑器。您可以将 Visual Studio 作为模块安装到现有的 Unity 安装中。有关更多信息，请参阅使用 Unity Hub 下载和安装编辑器和模块。

要手动设置默认的脚本编辑器：

  • 在主菜单中选择Edit > Preferences (macOS: Unity > Settings)以打开首选项窗口。
  • 打开外部工具菜单。
  • 单击外部脚本编辑器下拉菜单，并选择Microsoft Visual Studio。该选项的外观根据您安装的 Microsoft Visual Studio 的版本而有所变化。

A：外部脚本编辑器下拉菜单，显示所选脚本编辑器的名称和版本。

B：Unity 与所选脚本编辑器集成的 Unity 包的名称和版本。

Unity 使用 Visual Studio 的 C# 编译器来编译脚本。当您使用 Visual Studio 和 Visual Studio 编辑器包时，Unity 和 Visual Studio 都会显示脚本中的任何错误详细信息。

Unity 自动创建和维护 Visual Studio 的.sln 和.csproj 文件。您可以在首选项窗口的外部工具菜单中控制 Unity 是否为项目的某些元素生成.csproj 文件，如上面的屏幕截图所示。启用或禁用复选框以切换 Unity 是否为给定选项生成.csproj 文件。

在项目中的任何贡献者对文件状态进行更改（例如编辑现有文件或创建新文件）时，Unity 会重新生成项目的.sln 和.csproj 文件。您还可以从 Visual Studio 中向解决方案添加文件。Unity 会导入任何新文件，并在下次重新创建项目文件时将它们包含进去。
